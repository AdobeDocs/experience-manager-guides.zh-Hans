# 修复了2026.01.0版本中的问题

本文介绍Adobe Experience Manager Guides as a Cloud Service 2026.01.0版本中修复的各个方面的错误。

有关新功能和增强功能的更多信息，请查看 [2026.01.0 版本中的新增功能](whats-new-2026-01-0.md)。

了解2026.01.0版本[的](upgrade-instructions-2026-01-0.md)升级说明。

## 创作

- 使用上下文菜单中的编辑MathML选项更新内联MathML公式时，在刷新页面之前，更新后的值不会反映出来。 (指南 — 38198)
- 当主题包含许多可重用元素（具有ID的元素）添加到可重用面板时，由于容器高度固定，某些元素可能无法访问。 (指南 — 37220)
- 插入对文件的交叉引用时，映射和主题的图标相同。(指南 — 36662)
- 编辑地图时，`navtitle`中的特殊符号不会在“创作”视图中为`topichead`显示。 (指南 — 35435)
- 无法通过&#x200B;**另存为新版本**&#x200B;对话框向主题添加多个版本标签。 (指南 — 32716)

## 资产管理

- 无法从Assets UI的“版本历史记录”面板中移除版本标签。 (指南 — 38276)
- 上传文件名包含无效字符的资源时，资源上传失败并显示不正确的消息&#x200B;**文件被其他用户**&#x200B;锁定，尽管资源已解锁。 (指南 — 32680)
- 在Assets搜索中，子资源和元数据节点（如图像和PDF）错误地包含在结果中。 此外，对于应用DITAVAL筛选器时的输出预设，搜索将返回从条件预设创建的内部生成的DITAVAL文件。 (指南 — 35418)

## 发布

- 生成AEM Sites输出时，发布的输出中不包含包含关键字和主题标题以及`<ph>`元素的映射标题。 (指南 — 36641)
- 当使用包含指向PDF的链接的内容的自定义预设进行发布时，并且该范围未设置为外部时，流程将无法完成。 (指南 — 21708)
- 执行批量激活时，包创建会为页面的`fileReference`属性下列出的所有路径添加过滤器，包括外部路径和对等路径。 (指南 — 24887)
- 在本机PDF输出中，`abbreviated-form`元素显示`glossterm`，而不是指定的`glossSurfaceForm`或`glossAcronym`。 (指南 — 26393)
- 对于本机PDF输出，将忽略图像的`<alt>`元素，从而阻止将替换文本应用于辅助功能。 (指南 — 29087)
- 在发布预设期间下载带有基线的地图的临时文件时，`metadata.xml`文件错误地引用了`versionPath`而不是`dampath`。(指南 — 29815)
- 创建或编辑包含引文的主题时，如果未在引文对话框中添加“创作”字段，则不会生成PDF。 (指南 — 37934)
- CSS文件(`rhdefault.css`)被错误地应用于PDF模板，即使没有引用CSS，从而导致缺少CSS文件错误日志。(指南 — 31752)

## 平台

- 在尝试保存主题或映射时，操作可能会间歇性地失败，并出现&#x200B;**无法保存文件**&#x200B;错误，尤其是在后台运行密集型资源处理任务或翻译工作流期间。 (指南 — 37837)
- 在主题或映射中使用`scope="external"`引用DAM内容会导致资源的相对路径被替换为GUID。 (指南 — 38783)

## 报告

- 划分列表报告错误地包含当前映射范围内正确解析的外部链接、有效的`keyrefs`和关键字。 (指南 — 27774)

## 已知问题

Adobe发现了2026.01.0版本的以下已知问题：

- 从正在进行的审核任务中删除审核中的主题时，其文档状态继续保持为&#x200B;**在审核中**，即使该主题不再是任何审核任务的一部分。 (GUIDES-38709)<br>**解决方法**：从“属性”页面或“文件属性”面板，将主题的文档状态从&#x200B;**正在审阅**&#x200B;更改为适当的状态。
- 使用&#x200B;**查找并替换**&#x200B;执行搜索时，如果从搜索结果中打开文件，将其关闭，然后尝试通过选择列出的结果重新打开它，则无法重新打开文件。 (GUIDES-39050)<br>**解决方法**：首先从搜索结果中打开任何其他文件，然后从列表中重新打开之前关闭的文件以解决问题。
- 在将Guides与DB服务器结合使用时，对于包含自引用的内容，主题列表报告显示每个自引用的无效条目，从而导致文件计数不准确。 (指南 — 39420)
