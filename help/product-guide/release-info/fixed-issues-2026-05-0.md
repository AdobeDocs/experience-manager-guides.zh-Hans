# 修复了2026.05.0版本中的问题

本文介绍Adobe Experience Manager Guides as a Cloud Service 2026.05.0版本中修复的各个方面的错误。

有关新增功能和增强功能的详细信息，请查看[2026.05.0版本中的新增功能](whats-new-2026-05-0.md)。

了解2026.05.0版本](upgrade-instructions-2026-05-0.md)的[升级说明。

## 创作

- 使用&#x200B;**选择文件**&#x200B;对话框在编辑器中选择图像时，只显示光栅格式（如JPG、PNG和GIF）。 矢量文件(如.ai和.eps)不显示，也无法选择。 （指南 — 45110）
- 在升级时，`tagsView`设置默认关闭，即使它在`ui_config.json`中的默认值设置为`true`。 （指南 — 44651）
- 在编辑器中，文件引用显示为GUID，而不是文件路径，尽管配置了`xmleditor.uuid`。 （指南 — 42438）
- 当在DITA主题中应用具有相似键值的主题方案时，它们以几乎相同的颜色突出显示，这使得难以区分它们并识别应用了哪个方案。 （指南 — 38472）
- 在创作视图中编辑主题方案映射时，添加`hasInstance`元素会自动触发文件选择对话框，要求作者插入指向AEM资源的无需`href`。 此外，无法为此类映射加载&#x200B;**内容属性**&#x200B;面板，这会阻止作者更新创作视图中的元素属性，并要求他们使用Source视图来更新属性。 （指南 — 38164）
- 在编辑`.ditaval`文件时，当您切换到“创作”视图，然后返回到Source视图时，将删除在Source视图中添加的所有XML注释。 （指南 — 33228）


## 资产管理

- 在名称中包含空格的文件夹中创建主题时，会错误地创建一个重复的文件夹，其中空格被连字符替换，并且主题会保存在该文件夹中而不是原始文件夹中。 （指南 — 41938）
- 在首次翻译大型地图时，会为目标语言创建空XML文件，导致服务器负载增加和性能降低。 （指南 — 41613）
- 在基于数据库的环境中，有效的内部DITA链接在&#x200B;**资产属性**&#x200B;中显示为断开的链接，即使它们在编辑器和已发布的输出中正常工作。 （指南 — 35048）

## 发布

- 当对文件夹配置文件中输出预设的更改应用于现有映射时，为AEM Sites预设保存的&#x200B;**发布上下文**&#x200B;将被重置。 （指南 — 38377）
- 当在地图或书签的标题中使用`tm`元素时，商标符号(®)不会呈现在本机PDF输出的封面中。 （指南 — 28832）
- 使用本机PDF模板发布映射时，**映射标题**&#x200B;不包含映射`title`中使用的子元素中的内容，并且对应的内容筛选未应用于标题。(GUIDES-33730)
- 当AEM Sites输出中的交叉映射对等链接指向使用`chunk="to-content"`的`topicref`时，无法解析。 （指南 — 37873）
- 在发布期间，与基于DITAVAL的标记相关的文件将移动到新的系统生成的文件夹中，而不是保留在输出中它们预期的相对位置。 （指南 — 37564）
- 当使用具有冲突条件的多个DITAVAL文件时，原生PDF输出失败。 （指南 — 37484）

## 基准线

- 使用自定义DITA-OT时，映射中的主题引用错误地显示为间接引用，即使它们被直接引用。 此问题已使用新的基线体验得到解决。 （指南 — 19286）
- 具有`scope="peer"`的引用错误地包含在基线上下文中，从而导致发布时间比预期时间长。 此问题已使用新的基线体验得到解决。 （指南 — 30048）

## 平台

- 打开大型主题或地图时，创作实例会变得无响应，在某些情况下需要重新启动。 （指南 — 43547）

## 编辑器2.0中可用的已解决的问题

以下问题已修复，且在使用Editor 2.0（又称New Editor）时不再出现：

- 从表中删除两列或更多列时，表结构会变得不一致或损坏。 （指南 — 35438）
- 从包含合并单元格的表中删除列时，会添加新的空白列。 （指南 — 30147）
- 通过痕迹导航菜单在现有表中插入新行时，表结构会意外更改，导致缺少边框和额外的列。 （指南 — 29194）
- 在“创作”视图中，复制和粘贴表格行会将内容放置在表格之外，而不是将其作为表格中的新行插入。 （指南 — 24372）
- 在创作模式下使用鼠标拖动选择的节元素被复制并粘贴时，它将转换为段落`(<p>)`元素，而不是保留节结构。 （指南 — 30023）
- 编辑元素内的高亮显示文本（如step或uicontrol）时，所选文本无法正确替换，而是进行追加或追加，从而导致验证错误。 （指南 — 24371）
- 当使用相对值设置表列宽时，其余列不会按比例调整，从而导致表布局未对齐。 （指南 — 26109）
- 在禁用标记的情况下粘贴复制的主题标题时，第一次粘贴应用错误的样式，并将内容属性中的类型指定为主题而不是标题。 （指南 — 28838）
- 在编辑大部分内容时，意外的滚动移动会导致编辑器视图从活动内容中跳出。 （指南 — 35436）
- 在元素上使用Backspace时，无论光标位置如何，编辑器都会滚动到主题的顶部。 （指南 — 32520）
- 当使用向左或向右箭头键从内联标记中移出时，光标在第一次尝试时意外跳转。 （指南 — 26363）
- AEM Spellcheck仅适用于默认的en-US语言，不适用于其他区域设置。 （指南 — 14731）
- 在编辑器中解锁大型DITA主题时，同一主题将在重复选项卡中重新打开。 此外，还会触发标记限制警告，其中显示`NaN`而不是实际标记计数。 （指南 — 34008）
- 对于只读或锁定的主题，Acrolinx建议在编辑器中未正确高亮显示。 （指南 — 29614）
- 在创作视图中创建没有标题行的新`reltable`时，在第一个单元格中添加主题后，表布局会发生更改，导致下一列折叠并难以放置相关主题。 （指南 — 19555）
- 在“创作”模式下将`xref`链接添加到小型表单元格时，该链接不会保留在单元格内，而是会跨同一行的相邻单元格显示。 （指南–5489）
- 从创作视图切换到Source视图时，光标位置不会保留，并且编辑器会滚动回顶部。 此外，在长主题中，在Author视图和Source视图之间切换时，光标位置会丢失并随机跳转到中间或顶部。 （GUIDES-18114和GUIDES-21164）
- 在`<li>`元素内按&#x200B;*Enter*&#x200B;将创建新的`<li>`，但导航回前一个`<li>`并按&#x200B;*Enter*&#x200B;将错误地将当前项目的内容转换为`<p>`元素，从而破坏列表结构。 （指南 — 27505）

## 已知问题

Adobe发现了2026.05.0版本的以下已知问题：

- 当映射包含指向非DITA资源（如`.html`或`.htm`）的外部`topicref`时，其预览未显示在Assets UI中。 （指南 — 45409）
- 在Assets UI中，无法为DITA主题显示使用`conref`引用的内容，即使它在编辑器预览中正确呈现也是如此。 (GUIDES-45505)<br>**解决方法**：对于此类内容，您可以使用编辑器的预览。
- 启用`xmleditor.uniquefilenames`属性时，使用模板创建的新主题不包括主题标题。 （指南 — 44737）
