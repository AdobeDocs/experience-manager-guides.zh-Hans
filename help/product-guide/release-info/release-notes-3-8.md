# 发行说明 |Adobe Experience Manager Guides 3.8

**免责声明**：

*Adobe Experience Manager Guides*&#x200B;以前称为&#x200B;*XML Documentation for Adobe Experience Manager*。 请注意，文档中的某些引用可能仍引用以前的品牌，但仍适用于当前产品。

本发行说明列出了适用于Adobe Experience Manager的XML Documentation的3.8.x版本中的主要新增功能和增强功能。

## 3.8.5版本中的新增功能和增强功能

### 错误修复

3.8.5版本中修复的错误如下所示：

- 缺少通过FrameMaker Publishing Server输出的PDF的基线支持。
- 如果为AEM中的各个组设置了文件夹级别权限，则用于FrameMaker或Ourney的签出和签入API将无法正常工作。
- 不会从Assets UI页面显示内容预览。
- “Source”按钮在Assets UI页面上不起作用。
- 通过Web编辑器的“插入图像”功能插入图像时，插入图像的相对路径将更改为绝对路径。
- 最新的3.8版本中的UI中不显示FMPS预设下拉列表。
- 当DAM中包含大量资产并且从XML Web编辑器添加了新收藏项时，“收藏夹”面板不显示。
- 重定向所有链接的内部重定向&#x200B;*sling:mapping*&#x200B;不起作用，它显示网页的长URL（包含内部路径）而不是短URL。
- 在列表视图中，当从Assets UI页面（通过包管理器除外）上传或导入资源时，“已修改”列会显示“外部用户”而非用户名。
- 标题在地图仪表板的“主题”选项卡中无法正确显示。
- 启用节点拼合功能后，HTML输出中会存储一些不需要的垃圾字符。
- 对于已打开的文件，不会自动重新加载用户首选项中文件夹配置文件的更改，但需要刷新浏览器。
- 如果存在某些验证错误，则通过下载映射选项生成的输出会缺少一些主题。

## 3.8版本中的新增功能和增强功能

### 文件命名配置更新

在XML Documentation解决方案中创建DITA主题时，允许用户使用特殊字符作为文件名的一部分。 这会在生成AEM网站页面时生成经过编码的URL。 为避免在URL中进行此转换，3.8版本的XML Documentation解决方案允许管理员定义默认有效文件名配置(a-z A-Z 0-9 - _)之外的特殊字符列表。 这意味着，尽管您可以在包含空格的文件名中配置特殊字符列表，但该列表将被替换为连字符(-)。

### AEM网站页面名称生成更改

在创作时，不同文件夹下的一个或多个文件可能会具有相同的文件名。 在AEM网站发布过程中，当至少存在重复文件名时，会在页面名称后附加后缀。 在XML Documentation解决方案的3.8版本中，修复了AEM站点页面名称生成流程。 仅当存在重复的文件名时，才会将后缀附加到生成的页面名称。

### 新增功能和增强功能

本产品的以下方面引入了许多新增功能和增强功能。

#### Web编辑器

- 从Web编辑器创建主题版本时，现在可以从下拉列表中选择标签。

  下拉列表中的![标签](assets/labels-drop-down-saving-topic-res.avif)

- Web编辑器中的“审阅”面板现已功能更加强大，可让您将主题还原到共享以供审阅的版本。 您可以轻松地在审阅版本中加入审阅注释，而无需记住共享哪个版本的主题以供审阅。

  ![将主题还原为审阅版本](assets/revert-review-topic.avif)

- 引入了一个新的视觉提示来指示您使用的是主题的最新版本还是早期版本。

  ![Version提示](assets/old-version-icon.avif)

- 此发行版本中引入了新的版本历史记录功能。 使用“版本历史记录”功能可以：
   - 查看当前活动主题的所有版本的列表以及为每个版本添加的标签。
   - 还原到该主题的先前版本。

  ![版本历史记录](assets/version-history.avif)

- 已添加新的版本标签管理功能，该功能允许您将标签应用于主题的当前版本或早期版本。

  ![版本标签管理](assets/version-label-management.avif)

- 添加了新功能：“批准以供发布”，通过该功能，作者可以将资产标记为已批准，并进一步锁定资产以进行编辑。
- 在启动审阅流程时，您现在可以根据主题的状态筛选主题。

  ![根据其状态选择审核主题](assets/review-select-topic-on-state.avif)

- 使用添加到映射文件的主题标题自动填充映射中的`<navtitle>`。 您还可以从Web编辑器轻松地刷新`<navtitle>`。
- 现在，预览具有大量列的表会显示在页面区域中。
- 您可以从“属性”面板（多选）中一次应用多个输出类。
- 在预览主题时，您还可以直接从Web编辑器生成（无条件）单个主题的PDF输出。

  来自预览的![PDF输出](assets/pdf-output-from-preview.avif)

- 如果正在输出生成相同的预设，则阻止发布请求。
- 添加了仅允许一组特权用户删除具有活动后台引用的资源的功能。
- 添加了一项功能，用于从Assets UI的Source视图中查看或复制XML代码，即使文件已由其他用户签出。

  ![XML Source视图](assets/xml-source-view-from-assets-ui.avif)

- 文件名现在被替换为“保存”对话框、“可重用内容”面板和“查找并替换”面板中的文件标题。

#### 发布

- **允许为生成的站点页面配置清理规则**：作为管理员，您可以为生成的AEM站点或DITA-OT输出的文件名定义清理规则。 每当您使用DITA-OT生成AEM Site输出或输出时，都可以配置以下规则以清理输出生成的URL或文件名：
   - 将所有字符转换为小写。
   - 使用分隔符替换特殊字符。
   - 将长文件名限制为预定义的字符数。

- 使用批量激活仪表板，轻松地将输出从创作实例推送到发布实例。 您可以使用单个地图或地图收藏集，并选择要用于发布的输出预设。

  ![批量发布仪表板](assets/bulk-publish-dashboard.avif)

#### 性能改进

- AEM Sites输出的节点扁平化：早期AEM Sites输出的站点节点结构太深。 现在，您可以自行控制以扁平化节点结构以获得更好的性能。
- 支持最新版本的FrameMaker Publishing Server 2020年夏季版本。
- 现在将删除翻译过程中生成的临时文件，这会改进翻译过程。

#### 其他增强功能

- 删除了后处理DITA内容时DAM更新资产工作流的依赖关系。 如果在DAM更新资产工作流下定义了任何自定义流程步骤，则必须更新这些步骤，以便在后处理完成后执行。
- 现在，成功创建翻译作业后，翻译过程的发起者会在收件箱中收到通知。

### 错误修复

3.8版本中修复的错误如下所示：

- 音频对象不会显示在HTML输出中。
- 删除DITA主题时显示的强制删除窗口显示多个“强制删除”按钮。
- 将基线传输到语言副本对于使用服务器端视图创建的基线不起作用。
- 有时，主题版本3.0在并排视图中显示为3，不允许导入审阅注释。
- 在所移动DITA映射的“基线”选项卡上，引用内容详细信息的加载速度较慢。
- 还原到早期版本不适用于非DITA资产。
- 使用AEM Sites输出生成功能创建了许多空_text节点。
- XML编辑器 — 页面生成后图像映射中的链接解析不起作用。
- 从“基线”选项卡中应用标签不会将标签添加到引用的内容，如图像。
- 通过“下载映射”选项，以不正确的格式下载SVG文件。
- 无法在列表视图中编辑内容片段。
- 无法使用连接器在Oxygen XML Author中签出和打开文件。
- `<alt>`元素文本在“创作”视图中不可见。
- 即使存在翻译的副本，图像资产也始终显示为已过期。
- 自定义映射模板标题不正确，不显示缩略图。
- 应用的品牌标志元素不会显示在Web编辑器的创作视图中。
- 链接的脚注在内容中不可见。
- Web编辑器中的颜色编码不能用于特殊条件属性。
- @keyref的下拉列表对用户并不友好，因为它对于拥有大量@keyref的客户几乎不可用。
- @keyref引用的变量文本不会呈现。
- 氧气连接器 || “在氧气中编辑”按钮现在在编辑模式下打开文件，即使文件未签出。
- 未使用自定义映射模板创建自定义输出预设。
- Microsoft Word (.docx)到DITA的转换不会创建jcr:content节点，并允许为文件夹名称使用特殊字符。
- 移动MAP（引用超过150个）后，引用将中断，在打开主题时会观察到错误。
- 更改图像的宽度时，计算图像分辨率的方式不正确。
- 在`<codeblock>`中添加图像时，会在AEM站点输出中找到不需要的空格。
