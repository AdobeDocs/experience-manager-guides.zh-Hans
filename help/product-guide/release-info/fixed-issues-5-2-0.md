# 修复了5.2.0版本（2026年5月）中的问题

本文介绍5.2.0版本的Adobe Experience Manager Guides中修复的各个方面的错误。

有关新增功能和增强功能的详细信息，请查看[5.2.0版本中的新增功能](whats-new-5-2-0.md)。

了解5.2.0版本](upgrade-instructions-5-2-0.md)的[升级说明。


## 创作

- 如果将没有属性或值的空`prop`元素添加到DITAVAL文件中，则无法添加其他`prop`元素。 （指南 — 33597）
- 将Experience Manager Guides升级到2025.08.0版本后，**Workspace设置**&#x200B;中不再显示启用或禁用自定义&#x200B;**Acrolinx**&#x200B;选项卡的选项。 （指南 — 35261）
- 在浏览器刷新时，在主题或映射的文件夹级别配置文件中应用的自定义CSS将恢复到预览模式中的默认样式。 （指南 — 31098）
- **撤消**&#x200B;和&#x200B;**重做**&#x200B;操作在DITA文件编辑器的Source视图中无法按预期工作。 （GUIDES-24914和GUIDES-25034）
- 使用`Xref`元素在主题中引用DITA映射时，将显示引用的映射的文件名而不是映射标题。 （指南 — 21759）
- 通过编辑器界面的右侧面板添加多个Schematron文件进行验证时，即使添加的文件有效且没有错误，也会显示错误&#x200B;**Schematron文件不存在或存在错误**。 （指南 — 33731）
- 大型或复杂的MathML公式无法在编辑器中显示。 （指南 — 29095）
- 当打开多个DITA映射或主题并且其中一个主题关闭时，显示所有打开的选项卡的&#x200B;**>>**&#x200B;按钮与选项卡栏上其余打开的选项卡重叠。 （指南 — 31421）
- 启用&#x200B;**审批工作流**&#x200B;后，如果左侧面板和内容属性面板均打开，则编辑器工具栏上不会显示&#x200B;**开始新版本**&#x200B;按钮。 （指南 — 29093）
- 当代码片段名称超过代码片段面板的宽度时，它们会错误地封装到下一行中，从而导致与相邻的代码片段重叠并影响可读性。 （指南 — 22685）
- 当您将同一引用多次添加到DITA映射时，**映射**&#x200B;视图仅显示最后一次出现的标题，而以前的视图显示引用的UUID。 (GUIDES-9699)
- DITAVAL文件保持可编辑状态，即使它们被其他用户锁定，或者服务器启用了&#x200B;**禁用编辑，但不锁定文件**&#x200B;并以只读模式打开文件。 （指南 — 27754）
- 正在从内部清理作业生成缺少节点的日志，这些作业不是必需的，并且被错误地标记为错误，从而导致日志文件混乱。 （指南 — 31765）
- 在编辑架构(`*.sch`)文件并使用查找和替换功能时，“查找和替换”面板在底部部分显示在屏幕外，阻止访问其输入字段和控件。 （指南 — 38412）
- 无法从&#x200B;**另存为新版本**&#x200B;对话框向主题添加多个&#x200B;**版本标签**。 （指南 — 32716）
- 使用&#x200B;**选择文件**&#x200B;对话框在编辑器中选择图像时，只显示光栅格式（如JPG、PNG和GIF）。 矢量文件(如.ai和.eps)不显示，也无法选择。 （指南 — 45110）
- 在升级时，`tagsView`设置默认关闭，即使它在`ui_config.json`中的默认值设置为`true`。 （指南 — 44651）
- 在编辑器中，文件引用显示为GUID，而不是文件路径，尽管配置了`xmleditor.uuid`。 （指南 — 42438）
- 当在DITA主题中应用具有相似键值的主题方案时，它们以几乎相同的颜色突出显示，这使得难以区分它们并识别应用了哪个方案。 （指南 — 38472）
- 在创作视图中编辑主题方案映射时，添加`hasInstance`元素会自动触发文件选择对话框，要求作者插入指向AEM资源的无需`href`。 此外，无法为此类映射加载&#x200B;**内容属性**&#x200B;面板，这会阻止作者更新创作视图中的元素属性，并要求他们使用Source视图来更新属性。 （指南 — 38164）
- 在编辑`.ditaval`文件时，当您切换到“创作”视图，然后返回到Source视图时，将删除在Source视图中添加的所有XML注释。 （指南 — 33228）
- 使用上下文菜单中的编辑MathML选项更新内联MathML公式时，在刷新页面之前，更新后的值不会反映出来。 （指南 — 38198）
- 当主题包含许多可重用元素（具有ID的元素）添加到可重用面板时，由于容器高度固定，某些元素可能无法访问。 （指南 — 37220）
- 插入对文件的交叉引用时，映射和主题的图标相同。 （指南 — 36662）
- 编辑地图时，`navtitle`中的特殊符号不会在“创作”视图中为`topichead`显示。 （指南 — 35435）

## 资产管理

- 当您通过Experience Manager Guides UI重新上传编辑后的图像时，图像的原始演绎版会更新，但关联的DITA内容会继续显示图像的先前版本。 （指南 — 33693）

- 在Assets搜索中，子资源和元数据节点（如图像和PDF）错误地包含在结果中。 此外，对于应用DITAVAL筛选器时的输出预设，搜索将返回从条件预设创建的内部生成的DITAVAL文件。 （指南 — 35418）
- 上传文件名包含无效字符的资源时，资源上传失败并显示不正确的消息&#x200B;**文件被其他用户**&#x200B;锁定，尽管资源已解锁。 （指南 — 32680）

## 编辑器2.0中可用的已解决的问题

以下问题已修复，且在使用Editor 2.0（又称New Editor）时不再出现：

- 从表中删除两列或更多列时，表结构会变得不一致或损坏。 （指南 — 35438）
- 从包含合并单元格的表中删除列时，会添加新的空白列。 （指南 — 30147）
- 通过痕迹导航菜单在现有表中插入新行时，表结构会意外更改，导致缺少边框和额外的列。 （指南 — 29194）
- 在“创作”视图中，复制和粘贴表格行会将内容放置在表格之外，而不是将其作为表格中的新行插入。 （指南 — 24372）
- 在创作模式下使用鼠标拖动选择的节元素被复制并粘贴时，它将转换为段落`(<p>)`元素，而不是保留节结构。 （指南 — 30023）
- 编辑元素内的高亮显示文本（如step或uicontrol）时，所选文本无法正确替换，而是进行追加或追加，从而导致验证错误。 （指南 — 24371）
- 当使用相对值设置表列宽时，其余列不会按比例调整，从而导致表布局未对齐。 （指南 — 26109）
- 在禁用标记的情况下粘贴复制的主题标题时，第一次粘贴应用错误的样式，并将内容属性中的类型指定为主题而不是标题。 （指南 — 28838）
- 在编辑大部分内容时，意外的滚动移动会导致编辑器视图从活动内容中跳出。 （指南 — 35436）
- 在元素上使用Backspace时，无论光标位置如何，编辑器都会滚动到主题的顶部。 （指南 — 32520）
- 当使用向左或向右箭头键从内联标记中移出时，光标在第一次尝试时意外跳转。 （指南 — 26363）
- AEM Spellcheck仅适用于默认的en-US语言，不适用于其他区域设置。 （指南 — 14731）
- 在编辑器中解锁大型DITA主题时，同一主题将在重复选项卡中重新打开。 此外，还会触发标记限制警告，其中显示`NaN`而不是实际标记计数。 （指南 — 34008）

- 在创作视图中创建没有标题行的新`reltable`时，在第一个单元格中添加主题后，表布局会发生更改，导致下一列折叠并难以放置相关主题。 （指南 — 19555）
