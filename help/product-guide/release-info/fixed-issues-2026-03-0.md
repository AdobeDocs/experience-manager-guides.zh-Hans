# 修复了2026.03.0版本中的问题

本文介绍Adobe Experience Manager Guides as a Cloud Service 2026.03.0版本中修复的各个方面的错误。

有关新功能和增强功能的更多信息，请查看 [2026.03.0 版本中的新增功能](whats-new-2026-03-0.md)。

了解2026.03.0版本[的](upgrade-instructions-2026-03-0.md)升级说明。

## 创作

- 在编辑架构(`*.sch`)文件并使用查找和替换功能时，“查找和替换”面板在底部部分显示在屏幕外，阻止访问其输入字段和控件。 （指南 — 38412）

## 发布

- 当使用不同的条件预设在多个映射中重用同一主题时，将最新映射发布到Salesforce会覆盖主题内容，从而导致向以前发布映射的用户显示的数据不正确。 （指南 — 37806）
- 为包含条件处理或某些嵌套映射的映射发布本机PDF时，映射中定义的`dc:title`无法显示在PDF封面中，从而导致缺少封面标题。 （指南 — 37733）
- 为映射生成AEM站点输出（使用复合组件映射）失败，并在输出路径中存在`map_title`变量时导致错误。 （指南 — 36968）
- 当在本机PDF输出预设中指定了带尾随斜杠的输出路径时，UI会自动附加额外的尾随斜杠，从而导致在某些情况下导致PDF上传失败的双斜杠路径。 （指南 — 34932）
- 通过“快速发布”或“管理发布”发布从DITA内容生成的AEM Sites页面时，会无意中发布关联的DITA资产。 （指南 — 27967）
- 在将映射发布到AEM Sites（使用旧版组件映射）时，交叉引用(`xrefs`)具有`scope = peer`且在其他映射中针对主题的子元素（例如段落）时，交叉引用不会解析为特定的元素ID，而是只解析为父级主题，从而导致页面在主题的开头加载，而不是滚动到预期部分。 （指南 — 21802）


## 翻译

- 当最初作为具有特定版本（例如，在`/en/`下）的特定于语言的资产管理的图像移动到具有更新版本的全局文件夹并执行基线导出时，新基线将继续引用该图像的已过时的语言特定版本，从而导致基线导出失败。 （指南 — 39394）
- 处理在Experience Manager Guides外部创建的翻译项目时，会生成多个错误日志消息，以声明&#x200B;*`fmTarget`属性未找到*。 （指南 — 37804）

## 基准线

- 创建动态基线时，编辑器有时会因多个并发API请求而变得无响应，从而导致所有其他操作停止。 （指南 — 39054）

## 审阅

- 将用户分配给审核任务时，下拉列表会列出所有用户，而非仅列出与所选项目关联的用户，从而导致用户选项无效。 （指南 — 37781）

## 报告

- 打开地图的报表时，“筛选器”面板的加载存在延迟(GUIDES-39385)

## 已知问题

Adobe发现了2026.03.0版本的以下已知问题：

- 创建重复的ServiceNow知识库预设时，将加载空白屏幕。 （指南 — 42732）
- 用于检索文档状态的API对某些文件返回空响应，从而导致UI中显示不正确的文档状态。 （指南 — 42561）
- 未应用基于映射仪表板中选项卡名称的UI自定义项。 （指南 — 42285）
- 当文件在编辑器和搜索面板中打开时，从“资源管理器”面板中删除该文件会移除该文件并刷新“资源管理器”列表，但刷新页面会继续在“搜索”面板中显示该文件。 （指南 — 41935）
- 更新活动审阅任务时，如果删除了已属于审阅内容的主题，然后在不单击&#x200B;**更新**&#x200B;的情况下重新添加了主题，则“审阅者”选项卡中的审阅者信息将丢失。   （指南 — 38774）
- 如果主题位于bookmap标记（frontmatter、chapter、part或backmatter）内或部分循环内容中，则在预览模式下选择主题不会在“地图”视图中高亮显示该主题。 （指南 — 42416）
- 在Assets UI中，如果选择了超过2个文件或文件夹，则在第一次尝试时不会启用&#x200B;**移动**&#x200B;按钮。 (GUIDES-42721) <br> **解决方法**：选择两个以上的文件或文件夹后，等待几秒钟再选择目标文件夹。
- 当在编辑器中打开预览模式时从编辑器导航到&#x200B;**用户首选项**&#x200B;并更新根映射时，当您返回到编辑器时，映射预览将加载为空白屏幕。 (GUIDES-42412) <br> **解决方法**：使用在预览模式下可用的&#x200B;**刷新**&#x200B;图标刷新预览可解决此问题。
- 在手动刷新页面之前，重命名现有模板不会更新&#x200B;**输出模板**&#x200B;面板中的名称。 (GUIDES-42528)<br> **解决方法**：刷新浏览器以在输出模板面板中查看更新的模板名称。
